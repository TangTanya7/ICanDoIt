# ICanDoIt · 我能行 —— 产品方案（最终版）

> 一个基于行为心理学的目标达成平台。
> 不做内容，不做教学——只做一件事：**让你真的去做。**

---

## 一、产品定位

### 1.1 一句话定义

**ICanDoIt 是一个"让坚持自然发生"的行为驱动平台。**

它不生产知识，它让你真的去学。
它不教你怎么做，它让你真的去做。
它不绑架你坚持，它让坚持自然发生。

### 1.2 四大核心定位

```
ICanDoIt = 四根柱子撑起来的产品

  ① 欲望生成器 ——— 让你"想做"
  ② 温暖伙伴   ——— 让你"愿意听"
  ③ 无阻力触达 ——— 让你"马上做"
  ④ 规划导师   ——— 让你"做对的事"
```

| 定位 | 核心职能 | 对应心理学原理 |
|---|---|---|
| 欲望生成器 | 通过未来愿景、身份代入、进度可视化，持续激发用户对目标的渴望 | 心理对比、身份动机、损失厌恶 |
| 温暖伙伴 | 以有记忆、有性格的朋友身份陪伴用户全程，所有交互都有温度 | 自我决定理论（归属感）、自我慈悲 |
| 无阻力触达 | 在正确时刻通过最优渠道将任务送到用户面前，一点就到 | 执行意图、微习惯（B=MAT） |
| 规划导师 | 帮用户规划该做什么、去哪做，策展互联网优质资源排进每日任务 | 认知负荷理论、刻意练习 |

### 1.3 我们是什么，不是什么

```
❌ 我们不是内容平台（不做课程、不做教学）
❌ 我们不是待办清单（不是又一个 To-Do List）
❌ 我们不是打卡工具（不靠负疚感绑架用户）

✅ 我们是用户和他已有资源之间的「行为桥梁」
✅ 我们是一个有温度的「私人规划导师+督促者」
✅ 我们是一套经过心理学验证的「行为设计系统」
```

### 1.4 核心产品价值公式

```
用户的目标
+ 用户已有的资源（或系统推荐的优质免费资源）
+ 我们的行为设计（拆解 + 提醒 + 跳转 + 反馈）
= 目标达成
```

### 1.5 解决的核心问题

| 用户痛点 | 传统工具的失败 | ICanDoIt 的破解 |
|---|---|---|
| "目标太大，无从下手" | 用户自己拆解，认知负荷高 | AI + 模板自动拆解为每日 2-3 个微任务 |
| "三天热度，动力消退" | 靠意志力硬撑 | 未来愿景持续激发欲望 + 身份代入强化认同 |
| "忘记做，习惯断裂" | 闹钟式提醒，容易忽略 | 锚点绑定到日常习惯，搭便车触发 |
| "做了看不到效果" | 月底才看数据报告 | 0.3 秒内即时反馈 + 6 个维度的进度可视化 |
| "断签一次就放弃" | 连续天数归零，内疚→逃避→卸载 | 弹性机制 + 不惩罚 + 温暖复归 |
| "收藏了一堆资源但从来不看" | 没有人推你去用 | 帮你把收藏夹里的资源排进每日任务，到点提醒 |

### 1.6 目标用户

- **核心用户**：18-35 岁的自我提升者（学生、职场新人、斜杠青年）
- **典型场景**：想学英语/健身/学设计/阅读/理财，但总是半途而废
- **用户心理画像**：
  - "我知道我应该做，但就是坚持不下来"
  - "开头很兴奋，过一周就忘了"
  - "我缺的不是方法和资源，是持续行动的动力"
  - "收藏了一堆B站视频和课程，一个没看完"

## 二、心理学与行为学基础

ICanDoIt 的产品设计建立在 8 个经过实证验证的心理学理论之上。以下不是学术综述，而是每个理论在产品中的具体落地方式。

### 2.1 微习惯理论 —— 产品设计的「宪法」

**来源**：B.J. Fogg（斯坦福行为设计实验室）

**核心公式**：行为 = 动机 × 能力 × 触发（B = MAT）

这不是八个理论之一，而是统领所有设计的底层原则。ICanDoIt 的每一个功能都必须通过微习惯检验：

```
任务永远足够小 → 能力门槛足够低 → "不可能不做"
愿景持续激发   → 动机持续存在   → "一直想做"
锚点绑定触发   → 触发可靠且无阻力 → "自然就做了"

三者同时满足 → 行为自然发生 → 不依赖意志力
```

**2 分钟法则**：任何新习惯的启动版本必须能在 2 分钟内完成。先"进门"再"深入"——一旦开始做了，90% 的人会自然做更多（行为动量效应）。

| 目标 | 传统方式（高阻力） | 微习惯版本（低阻力） |
|---|---|---|
| 学英语 | 每天背 100 个单词 | 听 3 分钟对话 |
| 健身 | 每天跑 5 公里 | 做 5 个深蹲 |
| 阅读 | 每月读 4 本书 | 读 2 页 |
| 学设计 | 看完一整套课程 | 看一个 8 分钟的视频 |

### 2.2 心理对比 —— 驱动「未来愿景」

**来源**：Gabriele Oettingen（纽约大学，2012），WOOP 方法

**核心洞察**：单纯的积极幻想反而降低行动力（大脑误以为目标已实现）。但「先想象成功画面，再对比当前差距」能将目标达成率提升 **2-3 倍**。

**在产品中的落地**：每次打开 App → 先看未来愿景（3-5 秒沉浸）→ 紧接展示今日任务（对比→行动的无缝衔接）。

### 2.3 身份驱动 —— 驱动「身份代入系统」

**来源**：James Clear《Atomic Habits》+ Oyserman 身份动机理论（2007）

**核心洞察**：基于结果的目标（"我要学英语"）不如基于身份的目标（"我是一个持续学习的人"）持久。当行为与身份认同绑定后，放弃意味着否定自己——这是最强的坚持动力。

**在产品中的落地**：不只给称号标签，而是用叙事、日记、进化线让用户"感受到"身份在变化。

### 2.4 执行意图 —— 驱动「锚点绑定」

**来源**：Peter Gollwitzer（1999），If-Then Planning

**核心洞察**：将新行为绑定到已有习惯上（"如果刷牙，那么听英语"），可以绕过意志力直接触发行动。元分析显示效应量 d = .65。

**在产品中的落地**：锚点设定系统——不是定闹钟，而是将任务嫁接到用户已有的日常行为上。

### 2.5 操作性条件反射与变比率强化 —— 驱动「即时反馈」

**来源**：B.F. Skinner

**核心洞察**：行为后果决定行为重复概率。变比率强化（奖励不可预测但总会出现）是最强的行为维持机制。

**在产品中的落地**：
- 固定反馈：每次完成任务 → 进度更新 + 伙伴评语（保底安全感）
- 偶尔惊喜：伙伴在特殊节点的个性化鼓励、进度卡片自动生成（惊喜激发多巴胺）
- 关键：奖励在任务完成后 0.3 秒内呈现，绝不延迟
- 所有反馈用伙伴语气呈现，不是冷冰冰的系统通知

### 2.6 自我决定理论 —— 保护内在动机

**来源**：Deci & Ryan（2000）

**三大基本心理需求**：

| 需求 | ICanDoIt 如何满足 |
|---|---|
| 自主性 | 用户自定义目标/锚点/资源；弹性打卡机制；不强制每天全部完成 |
| 胜任感 | 连续天数、进度数据可视化、身份叙事日记、多维度回顾 |
| 归属感 | 伙伴角色的陪伴关系本身就是归属感；后期社交系统加强 |

### 2.7 自我慈悲 —— 驱动「反脆弱设计」

**来源**：Kristin Neff（2003），Adams & Leary（2007）

**核心洞察**：断签后对自己宽容（"没关系，每个人都有破例的时候"）比自我批评更能促进后续坚持。自我批评会导致"管他呢效应"——"反正已经破例了，干脆放弃"。

**在产品中的落地**：
- 断签后不归零、不惩罚
- 火焰变小但没灭
- 伙伴温暖迎接回归："你的树在等你。"
- 复归首日任务难度降低 50%
- 提供"主动休息日"机制

### 2.8 损失厌恶 —— 驱动「虚拟资产体系」

**来源**：Kahneman & Tversky 前景理论（1979）

**核心洞察**：人对"失去"的敏感度是对"获得"的 2 倍。

**在产品中的落地**：连续天数本身就是用户"不想失去"的东西——23 天的坚持记录，断签归零的心理成本远大于"今天不想做"的惰性。配合伙伴在断签时的温和拉回（"之前 23 天的努力还在"），既触发损失厌恶又避免内疚螺旋。

## 三、核心定位一：欲望生成器

让用户持续"想做"，而不是"应该做"。

### 3.1 未来愿景系统

**触发时机**：每次打开 App 时

**核心设计**：
- 根据用户的目标品类，生成具体的、感官化的未来成就场景
- 不是空洞的鸡汤（"你是最棒的！"），而是一个可以"看到"的画面
- 愿景内容至少每周刷新，防止疲劳
- 展示愿景后紧接展示今日任务——从"想象"无缝过渡到"行动"

**愿景示例**（英语学习目标）：

```
"你正在巴黎的咖啡馆里，
 用流利的法语与当地人畅聊，
 窗外是埃菲尔铁塔的轮廓。
 你笑着用当地俚语开了个玩笑，
 对面的法国朋友惊讶地说：
 '你的法语比很多法国人都好！'"

 ── 距离这个画面，你还有 73 天。
    今天的第一步：听一段 3 分钟的日常对话。
```

### 3.2 身份代入系统

不是给称号标签，而是让用户在每次互动中"感受到"自己正在成为另一个人。

**三个子机制**：

**① 身份叙事反馈**

每次完成任务时，不是冷冰冰的"已完成"，而是把具体行为和身份演进关联：

```
传统反馈：
  "✅ 已完成：学习 10 个新词汇"

身份代入反馈：
  "你刚刚又多掌握了 10 个词汇。
   现在的你，已经认识了 247 个高频词——
   这已经超过了大部分外企入门面试的词汇要求。
   你正在成为一个能用英语自由表达的人。"
```

**② 成长日记**

系统基于用户真实行为数据，在里程碑时刻自动生成第一人称视角的成长日记：

```
Day 30 的日记：
  "30 天。一个月前我按下了'开始'按钮，
   说实话，当时我并不确定自己能走到这里。
   但现在我已经掌握了 280+ 词汇，完成了 12 次口语练习，
   甚至开始能听懂一些英文播客的大意了。
   我已经不是那个'总是半途而废的人'了。
   我是一个说到做到的人。"
```

**③ 身份进化线**

一条叙事性的身份演进轨迹，不是冷冰冰的等级条：

```
[起点] 许下承诺的人
   ↓ Day 1-3
[第一次证明] 你证明了自己可以开始
   ↓ Day 4-7
[习惯萌芽] 新习惯开始扎根
   ↓ Day 8-21
[身份转换中] 这开始变成你的一部分
   ↓ Day 22-30
[新身份确立] 你已经是一个全新的人
   ↓ Day 31-90
[深度内化] 这不再是任务，这就是你的生活方式
```

每到达一个新阶段，触发沉浸式的身份确认仪式（全屏展示 + 回顾数据 + 伙伴深度对话）。

### 3.3 多维度进度激励

6 个时间维度的嵌套反馈环，让用户在任何时间尺度上，都"即将迎来下一次激励"。

| 维度 | 触发时机 | 形态 | 核心作用 |
|---|---|---|---|
| **每天** | 当日完成任务时 | 进度更新 + 伙伴评语 | 即时满足感 |
| **每周** | 每周日晚/周一早 | 周报卡片（伙伴帮你回顾语气） | 阶段成长感 |
| **每两周** | 每 14 天 | 习惯确认页 + 成长足迹卡片 | 习惯形成确认 |
| **每月** | 满 30 天 | 月度蜕变报告（全屏沉浸式） | 深度身份蜕变感 |
| **每季度** | 满 90 天 | 蜕变庆典（完整旅程回顾） | 习惯固化里程碑 |
| **每年** | 满一年/年底 | 年度成长年鉴（类似 Spotify Wrapped） | 人生记录感 |

**密度递减，重量递增**：

```
每天    ████████████████  频率高，单次轻量
每周    ████████████      频率中，有总结感
两周    ████████          频率中低，有确认感
每月    ██████            频率低，有仪式感
每季    ████              频率很低，有庆典感
每年    ██                频率极低，有纪念感
```

### 3.4 进度验证与自我对比

让用户 **亲眼/亲耳** 感受到自己的进步，比任何数据报表都有力：

- 录音对比：Day 1 的口语录音 vs Day 30 的录音
- 作品对比：Day 1 的设计作品 vs Day 30 的作品
- 数据对比：体重曲线、词汇量增长、阅读速度变化
- 眼光对比：Day 1 的审美测试成绩 vs Day 30 的成绩

用户主动记录（拍照/录音/输入数据），系统保存并在里程碑时刻呈现对比。

## 四、核心定位二：温暖伙伴

产品的灵魂层。**不是系统在给你发通知，是一个有记忆、有性格的老朋友在陪着你。**

### 4.1 角色人格定义

**它不是导师，不是教练，不是客服。它是你那个特别热忱、有点话痨、真心为你开心的老朋友。**

性格关键词：
- **热忱但不油腻**——真心为你每一点进步高兴，但不会说"你是最棒的！"
- **俏皮但知分寸**——会开玩笑，但你状态不好时会变温柔
- **记忆力好**——记得你第一天、你的里程碑、你上次什么时候来
- **诚实**——不会说"你做得完美"，会说"这个确实难，但你啃下来了"
- **随着关系变化**——Day 1 稍微客气，Day 30 称兄道弟，Day 100 像多年老友

### 4.2 语气随关系深度变化

| 阶段 | 天数 | 语气特征 | 示例 |
|---|---|---|---|
| 初识 | Day 1-7 | 温和、鼓励、稍正式 | "很高兴认识你，今天是你的第一天。走好每一步。" |
| 熟悉 | Day 8-30 | 轻松、偶尔俏皮 | "又来啦？我就知道你今天不会缺席。" |
| 老友 | Day 31-60 | 随意、敢开玩笑 | "哥们今天来得挺早啊？比太阳都积极。" |
| 战友 | Day 61-100 | 深度共鸣、提及共同回忆 | "还记得第15天你差点想放弃吗？但你回来了。从那天起我就知道你能走到今天。" |
| 灵魂伙伴 | Day 100+ | 简洁有力，无需多言 | "第147天了。不用我多说了吧？" |

### 4.3 各触点对话设计

**打开 App 时**（基于 6 个变量动态生成：距上次间隔、连续天数、时段、星期、状态、特殊节点）：

```
[正常每日，早晨]
"早啊！又是新的一天。
 今天是你坚持的第 47 天了——
 说真的，一个半月前刚认识你的时候，
 我还不确定你能走到这里。但你做到了。"

[几小时前刚来过]
"哥们！咱 4 小时前不是刚见过？
 你也太积极了吧，搞得我都有点不好意思了。"

[3天没来，回归]
"嘿！3 天没见了。
 我不打算问你去哪了，每个人都有自己的节奏。
 重要的是——你回来了。
 你之前 28 天攒下的底子还在，一个都没丢。
 今天轻松点，就做一个小任务热热身？"

[断签一周以上回归]
"……你回来了。
 之前积累的进度都还在，一个都没丢。
 很多人离开之后就再也没回来过。
 但你回来了。光这一点，就已经很了不起了。"

[第100天]
"一百天。
 你知道 100 天前你在干嘛吗？你在犹豫要不要按那个开始按钮。
 当时的你大概怎么也想不到，自己能连续做一件事 100 天吧？
 真的，这 100 天的你，太酷了。"

[深夜]
"嘿，这么晚了还来？
 我猜你白天太忙了？没事的，什么时候学都算数。
 晚上学的东西，睡一觉反而记得更牢。"
```

**完成任务时**：

```
[完成第一个任务]
"好的开始！今天的第一步已经迈出去了。
 剩下两个任务就是顺手的事。"

[完成挑战任务]
"这个挑战任务可不简单。
 很多人到这一步都会选择跳过。
 但你没有。我记住了。"

[全部完成，速度快]
"今天三个任务全部搞定！
 而且你今天的速度比上周快了——
 是不是觉得越来越轻松了？
 这就是进步的感觉。"
```

**困难检测时**（连续几天低完成率）：

```
"我注意到你最近两天都只完成了一个任务。
 不是在催你——能打开 App 已经赢了一半。
 如果最近确实有点累，这很正常。
 要不要我把任务调简单一些？"
```

### 4.4 反模板宣言（7 条原则）

1. **永远不说"亲爱的用户"**——你叫它什么？它叫你什么？得有专属称呼
2. **永远不说"恭喜你"**——真朋友会说"牛啊！"或更具体的话
3. **永远不说"继续加油"**——最空洞的中文，换成"明天见"或具体的下一步
4. **永远引用具体的事**——不是"你很棒"，而是"你已经学了280个词"
5. **根据关系深度变语气**——Day 1 温和，Day 30 敢开玩笑
6. **承认不好的状态**——不会说"你可以的！"，而是"今天不想做也没关系"
7. **用停顿和换行制造呼吸感**——好文案不是一大段，是有节奏的

## 五、核心定位三：无阻力触达

让"开始做"这件事的阻力降到接近零。

### 5.1 锚点设定

**核心理念**：不是定闹钟，是将新行为嫁接到已有的日常习惯上。

**设定方式**：对话式引导

```
伙伴："你每天早上大概几点起床？"
用户："7 点左右"
伙伴："那刷牙大概在？"
用户："7 点 10 分"
伙伴："好嘞！那以后每天 7:10 左右我提醒你，
      刷牙的时候顺便听 3 分钟英语？
      你可以随时调整，不合适我们再改。"
```

**锚点进化**：
- **第一层**：用户自报时间（MVP）
- **第二层**：从用户行为数据中学习优化推送时间
- **第三层**：系统级集成（闹钟/健康数据/日历，用户主动授权）

### 5.2 多渠道触达

| 渠道 | 优势 | MVP 优先级 |
|---|---|---|
| 微信订阅消息 | 到达率极高，用户习惯看微信 | P0 |
| App Push | 标准方式，支持富文本 | P0 |
| 短信 | 最强到达率 | P1 |
| 手机小组件（Widget） | 常驻桌面，无需推送 | P1 |
| 锁屏/灵动岛 | iOS 实时活动 | P2 |
| 智能手表 | 手腕震动提醒 | P2 |

**推送文案必须由伙伴引擎生成**：

```
❌ "叮！该学英语了"（命令式 → 抗拒）
✅ "刷牙了吗？今天的第一个任务只要3分钟——听一段关于点咖啡的对话 ☕"（融合式 → 自然）
```

### 5.3 深度链接一键直达

**2 步原则**：从看到提醒到开始做任务，最多 2 步。

```
收到推送/看到小组件 → 点击 → 直接在任务执行页了

不经过首页。不经过任务列表。不需要登录。
任务内容已经预加载好了。
打开就是开始——不需要"开始"按钮。
```

每条推送/消息都携带任务 ID 的深度链接，点击直达该任务。

### 5.4 实体锚点

**场景贴纸（MVP 轻量功能）**：

用户设定锚点后，系统生成一张可保存/打印的场景贴纸图片（含二维码）。

```
浴室镜子贴纸：  "🎧 刷着牙，听 3 分钟"
地铁卡旁贴纸：  "📱 坐下来，背 10 个词"
床头贴纸：      "🌙 睡前 5 分钟，复习今天学的"
电脑显示器旁：  "☕ 开工前，先做一个小练习"
```

二维码扫描后直达当日任务。

**NFC 触碰启动（P2）**：NFC 贴片贴在牙刷杯旁/门口/床头，手机碰一下直接启动任务。

### 5.5 零阻力启动设计

```
传统 App 的阻力链（7步）：
  收到推送 → 解锁手机 → 找到App → 打开App → 等加载 
  → 看到首页 → 找到任务 → 点击开始
  每一步都是一个"放弃点"

ICanDoIt 的目标（2步）：
  收到推送/看到小组件 → 点击 → 直接在任务里了
```

**小组件设计**：

```
┌────────────────────────────┐
│ 🔥 23天   🌱→🌿            │
│                            │
│ 📖 听一段点咖啡的对话       │
│ 只要 3 分钟               │
│                            │
│      [ ▶ 现在开始 ]        │
└────────────────────────────┘

显示今天最该做的那一个任务 + 一键开始
随时间变化文案：
  早晨："早上好！今天第一个任务准备好了 ☀️"
  下午（还没做）："今天还没开始呢？来一个？"
  晚上（全做完了）："今日满分 ✨ 明天见！"
```

### 5.6 渐进脱离提醒机制

**终极目标：让用户不需要提醒也能自动行动。**

```
第 1-14 天（依赖期）：
  每个锚点时间主动提醒

第 15-30 天（过渡期）：
  提醒频率降低，用户如果不被提醒也主动来了，伙伴说：
  "今天没提醒你，你自己就来了！
   你的大脑已经在建立新的自动回路了。"

第 31-60 天（独立期）：
  基本不在锚点时间提醒，只在用户"忘记"时轻推

第 60 天+（内化期）：
  用户连续 14 天不需要提醒就主动完成
  → 解锁特殊成就"习惯独立勋章"
  → 伙伴说："你已经不太需要我了。
     但我还是会在这里，看着你继续往前走。"
```

## 六、核心定位四：规划导师

帮用户"做对的事"——知道该学什么、去哪学、按什么顺序学。

### 6.1 产品定位：策展+编排，不生产内容

```
我们不生产内容，我们是内容的"导游"：

  · 知道你该学什么 → AI + 领域知识
  · 知道去哪学最好 → 资源策展 + 社区推荐
  · 知道按什么顺序 → 学习路径编排
  · 知道怎么练     → 可执行的练习指令
  · 知道怎么验证   → 阶段测试 + 自我对比
```

### 6.2 目标解析：对话式拆解

用户只需回答不超过 3 个问题，系统完成全部拆解：

```
伙伴："版式设计！好方向。简单问两个就开始——
      你现在大概什么水平？"
      □ 纯小白
      □ 知道一点，但不系统
      □ 有基础，想提高

伙伴："每天愿意花多少时间？"
      □ 10分钟 □ 20分钟 □ 30分钟+

伙伴："够了！我给你排好了30天的计划。今天先从最基础的开始——"
```

**支持同时最多 3 个目标**，每个目标独立路径和锚点。

**提供热门目标模板**（一键选择）：
- "30天减重10斤"
- "30天养成阅读习惯"
- "30天入门UI设计"
- "考研倒计时100天"
- "30天存下3000块"

### 6.3 路径编排：学+练+反馈

每日 2-3 个任务，三种类型混合：

| 类型 | 行为 | 示例 |
|---|---|---|
| **学习（输入）** | 看视频/听播客/读文章 | "去B站看这个8分钟的网格系统讲解" |
| **练习（输出）** | 动手做/模仿/创作 | "打开Figma，用网格排版一张海报" |
| **反馈（确认）** | 记录/自评/拍照 | "把你做的排版截图发上来" |

**配比随阶段递进**：

```
前期（建立认知）：学 50% + 练 30% + 反馈 20%
中期（开始动手）：学 30% + 练 50% + 反馈 20%
后期（独立创作）：学 10% + 练 70% + 反馈 20%
```

### 6.4 资源策展：三层筛选

```
第一层：AI 初筛（效率）
  → 播放量/收藏量/评分等硬指标过滤

第二层：用户投票（真实）
  → 做完后回答"这个资源有帮助吗？" 👍 / 👎 / 换一个
  → 高赞上浮，差评替换

第三层：运营审核（兜底）
  → 热门目标的核心资源人工确认质量
```

**每个任务的资源选择权**：

```
默认推荐一个最优资源（零思考，直接用）
旁边有"换一个"按钮 → 展示 2 个备选 + 自填链接
用户不点"换"就不需要做任何选择
```

### 6.5 动态调整：反馈驱动

**行为信号 → 调整动作**：

| 信号 | 调整 |
|---|---|
| 连续 3 天秒完成 | 伙伴提议加难度 |
| 连续 3 天只完成 1 个 | 伙伴关心，主动降低任务量 |
| 某类任务总被跳过 | 降低该类型频率或难度 |
| 资源被评差评 | 自动替换为备选资源 |
| 用户偏好视频 | 后续优先推荐视频 |

**每周轻量回顾**（3 题问卷）：

```
伙伴："这周过得怎么样？快速说一下——"

  这周的内容对你来说：□ 太简单 □ 刚好 □ 有点吃力
  你更喜欢哪种学习方式：□ 看视频 □ 看文章 □ 直接动手做
  这周最有帮助的是哪个任务？[选择]

→ 根据回答调整下周计划
```

### 6.6 完成验证

**MVP**：轻量验证——跳转型任务回来后，回答 1 道 10 秒内能完成的选择题，既验证又巩固学习。

**V2**：增加时间检测——任务预计 10 分钟但 30 秒就回来，伙伴温和提示。

**反作弊递进惩罚**（保持伙伴语气）：
1. 首次：温和提醒——"我觉得你好像没太认真看？再看一遍可能效果更好"
2. 多次：连续天数不计入——"如果没真的完成，今天就先不算了？"
3. 持续：伙伴诚实对话——"我注意到最近好几次都跳过了……要不咱们聊聊，是不是任务不太合适？"

## 七、进度与坚持机制

> 设计原则：我们是效率工具，不是游戏。用户打开 App 是为了做事，不是为了养东西。激励来自「看到自己真的在进步」，而不是虚拟奖励。

### 7.1 连续天数

```
🔥 23 天
```

一个数字，干净直接。
- 断签 → 数字重置，但伙伴会说"之前 23 天完成的 68 个任务不会消失"
- 休息日 → 不算断签（见 7.3）
- 不搞火焰等级、不搞倍率加成，就是一个诚实的计数器

### 7.2 进度记录 —— 里程碑时刻的自动记录

在关键节点自动生成一张"进度卡片"，永不消失：

```
Day 7:   "第一周结束，你完成了 15 个任务"
Day 14:  "两周了，累计完成 34 个任务，最常用的资源是B站"
Day 30:  "一个月——68 个任务，坚持率 82%"
Day 60:  "两个月的你 vs 第一天的你：[具体数据对比]"
Day 90:  "90 天旅程完成。回看你走过的路。"
```

这些卡片存在"我的进度"里，用户随时可以回看。也是分享到社交平台的素材——分享的是真实的进步数据，不是虚拟成就。

### 7.3 休息日

```
每周自动有 1 天可以标记为"休息日"
不需要购买，不需要赚取
标记后：连续天数不中断
伙伴说："休息也是坚持的一部分。明天见。"
```

### 7.4 即时反馈

任务完成后的反馈保持简洁：

```
完成一个任务：
  ✅ 打勾动画 + 进度更新（"今日 2/3"）+ 伙伴说一句话

全部完成：
  🎯 "今日全部完成" + 伙伴的一句具体鼓励

反馈的核心是伙伴那句具体的话，不是特效。
```

### 7.5 设计边界：不做什么

我们是效率工具，以下这些**明确不做**：

| 不做的 | 理由 |
|---|---|
| 成长树/虚拟宠物 | 养成类机制是游戏思路，偏离效率工具定位 |
| XP / 经验值 | 数字堆积感太"游戏" |
| 等级 / 称号 | 刷等级不等于完成目标 |
| 虚拟货币 | 引入了不必要的经济系统 |
| 成就徽章 | 收集成就本身不帮助用户完成目标 |
| 变比率奖励 | 老虎机机制不适合效率工具 |

**判断标准**：这个功能是帮用户把事做了，还是让用户在"做事"之外多了一件事？只留前者。

## 八、产品路线图

### Phase 1 — MVP（4-6 周）

**目标**：验证核心假设——「欲望生成器 + 温暖伙伴 + 无阻力触达 + 规划导师」四柱模型能否提升用户留存和目标达成率

| 模块 | MVP 范围 |
|---|---|
| 用户账号 | 微信一键登录 + 简版引导 |
| 目标设定 | 3 个问题对话式设定 + 5-10 个热门模板；支持同时 3 个目标 |
| 欲望引擎 | 预设愿景文案库 + 每日展示 |
| 路径规划 | 热门目标预设路径 + 人工策展资源库 |
| 锚点触达 | 用户自选锚点 + 微信消息推送 + 深度链接 + 场景贴纸 |
| 每日任务 | 任务卡片 + 外部跳转 + 轻量验证 + 换资源 |
| 伙伴引擎 | 文案模板库 + 变量插值，覆盖核心触点 |
| 进度追踪 | 每日/每周回顾卡片 + 基础数据面板 |
| 坚持机制 | 连续天数 + 进度记录 + 休息日 |
| 动态调整 | 每周回顾问卷（3题）→ 半自动调整 |
| 平台 | 微信小程序优先 |

### Phase 2 — 增长版（8-12 周）

- 接入 AI 大模型：智能任务拆解 + 个性化愿景生成 + 伙伴完全动态对话
- 身份代入系统完善：身份日记、身份进化线
- 社交系统：好友/组队/互相督促
- 智能推送：基于行为数据优化推送时间
- 多维度回顾：月度蜕变报告、季度庆典
- NFC 贴片实体锚点
- 进度数据更丰富的可视化（热力图等）
- 时间检测验证 + 反作弊机制
- 小组件（Widget）支持

### Phase 3 — 成熟版（6 个月）

- 多端上线：iOS / Android / 鸿蒙
- 伙伴人格可选：温柔型/毒舌型/搞笑型等
- 内容市场：用户/达人创建并分享学习路径
- 社区 & 排行榜
- 年度成长年鉴
- 商业化：会员 / 高级 AI 功能 / 内容付费
- 第三方集成：日历/健康数据/智能家居联动
- 目标上限拓展

## 九、核心指标

| 指标 | 定义 | 目标值 |
|---|---|---|
| **注册转化率** | 进入→完成注册+目标设定 | > 60% |
| **D1 留存率** | 注册次日仍活跃 | > 50% |
| **D7 留存率** | 注册 7 天后仍活跃 | > 40% |
| **D30 留存率** | 注册 30 天后仍活跃 | > 20% |
| **日任务完成率** | 每日至少完成 1 个任务的用户比例 | > 60% |
| **日全勤率** | 每日完成全部任务的用户比例 | > 30% |
| **锚点响应率** | 推送后 30 分钟内打开的比例 | > 35% |
| **资源好评率** | 推荐资源获得 👍 的比例 | > 70% |
| **目标完成率** | 在设定周期内完成目标的比例 | > 15%（行业均值 < 8%） |
| **习惯独立率** | 达成"不需要提醒也主动完成"的用户比例 | > 10% |
